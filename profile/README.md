## Netflix 的检测逻辑，搞懂它才能选对机房

Netflix 的封锁本质是 IP 黑名单机制。他们维护着一个已知数据中心 IP 段的列表，一旦检测到你用的 IP 属于这个范围，直接拒绝播放——哪怕你那个 IP 今天还能用，过了一段时间可能就被加进去了。

所以搬瓦工哪个机房可以看 Netflix，不存在一个永久有效的答案，只有"目前相对概率更高的机房"。根据多方实测和用户长期反馈，目前比较靠谱的有以下几个方向：

**弗里蒙特机房（USCA_FMT）**
美国加州弗里蒙特，相比洛杉矶几个 CN2 机房，这里的 IP 段被 Netflix 封锁的程度相对低一些。历史上 `97.64.36.*` 是已知可用的 IP 段之一。这个机房走的是 HE（Hurricane Electric）线路，不是 CN2 GIA，对国内速度不是最好，但如果你的核心需求就是看 Netflix，它是当前相对务实的选择。

**洛杉矶 DC4 MCOM 机房（USCA_4）**
DC4 是另一个被用户长期测试的机房，部分 IP 段的封锁程度比 DC6 低，可以通过 Netflix 检测。不过具体能不能用，还是要看你分配到哪个 IP。

**日本软银机房（JPOS_1）**
这个机房是很多用户的惊喜——日本软银（SoftBank）线路，IP 段相对"干净"，被 Netflix 列入黑名单的比例相对低。重要的是，日本机房对应的 Netflix 日区内容非常丰富，动漫、日剧资源是其他地区比不了的。速度上对国内用户也很友好，联通走软银，电信走 CN2 GIA，整体延迟很低。流媒体需求大的用户，日本软银是目前搬瓦工阵列里最值得优先试的机房之一。

**荷兰 AS9929 机房（EUNL_9）**
荷兰联通 9929 线路，IP 段同样相对干净，对 Netflix 欧区内容有一定解锁能力。喜欢看英国 BBC 内容或欧洲剧的用户可以关注。

## 具体操作：买了之后怎么试

1. **购买套餐后，登录 KiwiVM 控制面板**，记下当前 IP。
2. **打开 Netflix**，直接访问，看是否出现"检测到代理"提示。如果可以正常播放，恭喜你运气不错。
3. **如果不行**，进入 KiwiVM 的"Migrate to another DC"功能，切换到弗里蒙特或日本软银机房，换一个新 IP，再试一次。
4. 重复上述过程，直到找到可用的 IP。
5. **一旦找到可用 IP，尽量不要主动更换**，保持稳定使用。

需要特别注意的一点：**不要在 iOS 的 Netflix 官方 App 上测试**。有用户反馈，iOS App 有额外的 DNS 查询检测机制，会导致 IP 被 Netflix 拉黑 48 小时。建议优先在浏览器里测试。
